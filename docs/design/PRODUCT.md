# chatpress_V1 产品定义

## 1. 产品概述

chatpress_V1 是一个面向 AI 内容沉淀场景的轻量级知识页面发布系统。

系统允许用户将 AI 对话整理稿或 Markdown 笔记保存为结构化内容，并发布为可访问、可分享的网页。第一版聚焦最小可用发布链路：内容输入、持久化、Markdown 渲染和公开页面访问。

## 2. 产品定位

chatpress_V1 面向介于 AI 聊天工具、笔记工具、博客系统和静态站点生成器之间的轻量发布场景。

AI 聊天工具适合生成内容，但不适合长期组织和发布内容。传统笔记工具适合记录，但公开分享能力有限。博客系统和静态站点生成器具备发布能力，但对零散知识笔记和 AI 对话整理稿来说流程偏重。

chatpress_V1 的定位是：

> Turn AI chats and Markdown notes into shareable knowledge pages.

中文定位：

> 把 AI 对话和 Markdown 笔记变成可分享的知识页面。

## 3. 目标用户

第一版主要面向以下用户：

- 需要整理课程笔记、复习材料和学习总结的学生。
- 需要沉淀技术笔记、调研记录和项目文档的开发者。
- 高频使用 ChatGPT、Claude、Gemini、Cursor 等 AI 工具生成内容的用户。
- 需要以比博客和静态站点更低成本方式发布知识页面的个人用户。

第一版不面向通用博客写作者、社交媒体创作者或多人协作团队。

## 4. 核心场景

用户在 AI 工具、本地 Markdown 文件或个人笔记中已经积累了有价值的内容，但这些内容通常缺少稳定的保存结构和分享入口。

chatpress_V1 的核心场景是：

```text
用户整理一段 AI 对话或 Markdown 笔记
-> 在系统中创建 Artifact
-> 系统保存原始内容并渲染 HTML
-> 用户通过公开 URL 访问和分享页面
```

## 5. MVP 目标

MVP 的目标是验证完整的内容发布链路：

1. 用户可以创建一篇 artifact。
2. 系统可以保存原始 Markdown 内容。
3. 系统可以生成 HTML 渲染结果。
4. 用户可以通过公开 URL 查看页面。
5. 用户可以对已有 artifact 进行编辑、查询和删除。

当用户能够将一段 AI 对话整理稿或 Markdown 笔记发布为可阅读页面时，MVP 即达到目标。

## 6. MVP 功能范围

第一版必须包含：

- 创建 artifact。
- 编辑 artifact。
- 删除 artifact。
- 查看 artifact 列表。
- 查看 artifact 详情。
- Markdown 渲染为 HTML。
- 基于 slug 的公开页面访问。

第一版可以包含：

- 简单默认页面样式。
- 基础字段校验。

## 7. 非目标

以下能力不属于 MVP 范围：

- 自动读取 ChatGPT、Claude、Gemini、Cursor 的聊天记录。
- LLM 集成。
- AI 可读 JSON 或文本导出。
- MCP server。
- DOCX 转换。
- PDF 解析。
- 语义搜索。
- 知识图谱。
- 评论系统。
- 用户账号。
- 权限系统。
- Redis。
- 插件市场。
- 浏览器扩展。
- VS Code 或 Cursor 扩展。
- 多用户协作。
- 复杂部署自动化。
- 托管 SaaS 功能。

这些能力可作为后续版本方向，但不进入第一版实现范围。

## 8. 首个用户流程

```text
打开应用
-> 创建新的 artifact
-> 输入标题
-> 粘贴 AI 对话整理稿或 Markdown 内容
-> 发布
-> 打开公开页面 URL
```

公开 URL 示例：

```text
/p/my-java-note
```

## 9. 核心数据对象

第一版使用一个核心对象：`Artifact`。

`Artifact` 表示一篇可发布的知识页面，包含以下信息：

- 标题。
- Slug。
- 原始格式。
- 原始内容。
- 渲染后的 HTML。
- 状态。
- 创建时间。
- 更新时间。

## 10. 产品原则

chatpress_V1 第一版遵循以下原则：

- 保持功能边界清晰。
- 优先打通完整发布链路。
- 先支持手动整理后的 Markdown 输入。
- 避免在 CRUD、渲染和公开访问完成前引入复杂基础设施。
- 为后续 AI-native 能力保留扩展空间，但不让其阻塞 MVP。

## 11. 后续方向

MVP 稳定后可以评估：

- AI 聊天记录导入。
- HTML 内容发布。
- 更好的页面主题。
- 标签和搜索。
- Markdown / Text / JSON 导出。
- `llms.txt`。
- LLM 生成摘要和标签。
- DOCX 导入。
- 静态站点导出。
- 自定义域名。
- 托管版本。
