# ChatPress 产品定义

## 1. 产品概述

ChatPress 是一个轻量级、开源的知识页面发布系统。

它帮助用户把 AI 聊天记录和 Markdown 笔记整理成干净、可分享、可长期保存的个人知识页面。

第一版只关注一个简单的发布流程：

```text
写入一篇 Markdown 笔记，或粘贴 AI 对话整理内容
-> 发布
-> 得到一个干净的公开页面
```

ChatPress 第一版不打算做成完整博客平台、Notion 替代品，或完整的 AI 知识管理系统。

## 2. 产品定位

ChatPress 位于几类产品之间：

- 传统博客：通常更偏文章发布，对普通知识笔记不够轻。
- Notion 类工具：适合编辑，但不一定适合发布结构清晰、可分享的知识页面。
- 静态网站生成器：功能强大，但对很多用户来说太偏工程化。
- AI 聊天工具：适合生成内容，但聊天记录容易分散，不适合长期发布和复用。

产品定位是：

> Turn AI chats and Markdown notes into shareable knowledge pages.

中文定位：

> 把 AI 对话和 Markdown 笔记变成可分享的知识页面。

## 3. 目标用户

第一版面向一个较窄的用户群：

- 想发布课程笔记、总结、复习材料的学生。
- 想发布技术笔记和项目文档的开发者。
- 经常用 ChatGPT、Claude、Gemini、Cursor 等工具生成聊天记录、学习总结或 Markdown 笔记的 AI 重度用户。
- 想要比完整博客或静态网站生成器更简单的发布流程的用户。

第一版不面向泛博客作者或社交媒体创作者。

## 4. 核心问题

很多用户已经有有价值的 AI 聊天记录、Markdown 笔记或 AI 生成文本，但这些内容经常散落在聊天记录、本地文件或私有工作区里。

核心问题是：

> 用户需要一种简单方式，把有价值的 AI 对话和 Markdown 笔记变成干净、持久、可分享的网页。

ChatPress 应该先解决这个问题，再考虑更高级的 AI、搜索、协作或部署能力。

## 5. MVP 目标

MVP 要证明用户可以完成：

1. 创建一篇笔记。
2. 保存这篇笔记。
3. 把笔记渲染成干净的 HTML 页面。
4. 通过公开 URL 查看笔记。
5. 后续编辑或删除笔记。

如果用户可以把一段 AI 对话整理成 Markdown 内容并发布为可阅读页面，或直接发布一篇 Markdown 笔记，就说明 MVP 成功。

## 6. MVP 功能

第一版必须包含：

- 创建笔记。
- 编辑笔记。
- 删除笔记。
- 列出笔记。
- 查看笔记详情。
- 把 Markdown 渲染成 HTML。
- 为每篇笔记展示一个公开页面。
- 支持用户粘贴从 AI 聊天中整理出来的 Markdown 内容。

第一版可以包含：

- 基于 slug 的 URL。
- 一个非常简单的默认页面样式。

## 7. MVP 非目标

以下功能明确不属于第一版范围：

- AI 可读的 JSON 或文本导出。
- LLM 集成。
- 自动读取 ChatGPT、Claude、Gemini、Cursor 的聊天记录。
- MCP server。
- DOCX 转换。
- PDF 解析。
- 语义搜索。
- 知识图谱。
- 评论系统。
- 用户账号。
- 权限系统。
- Redis。
- 插件市场。
- 浏览器扩展。
- VS Code 或 Cursor 扩展。
- 多用户协作。
- 复杂部署自动化。
- 托管 SaaS 功能。

这些功能未来可以考虑，但不应该在基础发布流程可用之前实现。

## 8. 第一个用户流程

第一版的用户流程应该是：

```text
打开应用
-> 创建一篇新笔记
-> 输入标题
-> 粘贴 AI 对话整理后的 Markdown 内容
-> 发布
-> 打开公开页面 URL
```

公开 URL 示例：

```text
/p/my-java-note
```

## 9. 第一个数据对象

第一个核心对象是 `Artifact`。

一个 artifact 表示一篇已发布的知识页面。

在 MVP 阶段，一个 artifact 应该包含：

- 标题。
- Slug。
- 原始格式。
- 原始内容。
- 渲染后的 HTML。
- 状态。
- 创建时间。
- 更新时间。

## 10. 产品原则

ChatPress 应该遵守这些原则：

- 保持第一版足够小。
- 优先完成一个完整发布流程，而不是堆很多半成品功能。
- 先让产品在没有 AI 的情况下有用。
- 在核心 CRUD 流程跑通前，不添加高级基础设施。
- 把 AI 对话整理后的 Markdown 发布作为第一个实际使用场景。
- 保留未来 AI-native 功能的可能性，但不让它们成为 MVP 的必要条件。

## 11. 未来可能方向

MVP 可用之后，项目可以继续探索：

- HTML 笔记发布。
- AI 聊天记录导入。
- 更好的主题。
- 标签和搜索。
- Markdown / Text / JSON 导出。
- `llms.txt`。
- LLM 生成摘要和标签。
- DOCX 导入。
- 静态站点导出。
- 自定义域名。
- 托管版本。

这些方向应该在第一版可用后再评估。
